## 1. 셀러 건강 점수 (Health Score)

### 개요

셀러의 전반적인 경영 상태를 **0~100점** 사이의 단일 점수로 요약합니다. 6개 차원을 개별 점수화한 뒤 가중 합산합니다.

### 6차원 가중치

| 차원 | 가중치 | 설명 |
|------|--------|------|
| **매출 (revenue)** | 20% | 총 매출 규모 |
| **주문량 (orders)** | 15% | 총 주문 건수 |
| **리뷰 (review)** | 25% | 평균 리뷰 점수 + 저평가 비율 |
| **배송 (delivery)** | 20% | 평균 배송일 + 배송 지연율 |
| **상품 (product)** | 10% | 상품 다양성 (고유 상품 수) |
| **고객도달 (reach)** | 10% | 고유 고객 수 |

> 리뷰(25%)와 배송(20%)에 가장 높은 가중치를 부여한 이유: 상관분석 결과 리뷰 점수와 배송 품질이 매출·재구매에 가장 큰 영향을 미치는 것으로 확인되었기 때문입니다.

### 차원별 점수 계산 방법 (각 0~100점)

#### 매출 점수
```
매출 점수 = clamp( log(1 + 셀러매출) / log(1 + Top매출) × 100 )
```
- Top Performer 매출(R$12,161) 대비 **로그 스케일 비율**
- 로그를 사용하는 이유: 매출 분포가 극도로 편향(소수의 고매출 셀러)되어 있어, 선형 비율로는 대부분의 셀러가 0점 근처에 몰리기 때문

#### 주문량 점수
```
주문량 점수 = clamp( log(1 + 셀러주문수) / log(1 + Top주문수) × 100 )
```
- Top Performer 주문수(98.5건) 대비 로그 스케일 비율
- 매출과 동일한 이유로 로그 적용

#### 리뷰 점수
```
기본점수 = (평균리뷰 - 1) / 4 × 100      # 1점→0, 5점→100
패널티 = 저평가비율 × 50                   # 1~2점 리뷰가 많으면 감점
리뷰 점수 = clamp(기본점수 - 패널티)
```
- 평균 리뷰(1~5점)를 0~100 스케일로 변환
- **저평가 패널티**: 리뷰 1~2점 비율이 높으면 추가 감점 (예: 저평가 30% → -15점)

#### 배송 점수
```
배송일 점수 = clamp( (30 - 평균배송일) / 23 × 100 )    # 7일=100, 30일=0
지연율 점수 = clamp( (1 - 지연율) × 100 )               # 0%=100, 100%=0
배송 점수 = 배송일점수 × 0.6 + 지연율점수 × 0.4
```
- 배송일 7일 = 만점, 30일 = 0점의 선형 스케일
- 배송일(60%)과 지연율(40%)을 혼합

#### 상품 점수
```
상품 점수 = clamp( 셀러상품수 / Top상품수 × 100 )
```
- Top Performer 상품 다양성(36.3종) 대비 선형 비율

#### 고객도달 점수
```
고객도달 점수 = clamp( log(1 + 셀러고객수) / log(1 + Top고객수) × 100 )
```
- Top Performer 고객수(87.4명) 대비 로그 스케일 비율

### 최종 건강 점수
```
건강 점수 = Σ(차원 점수 × 가중치)  # 0~100으로 클램프
```

### 등급 분류

| 점수 범위 | 등급 | 대시보드 표시 |
|-----------|------|--------------|
| 80~100 | 우수 | 녹색 / "현 수준 유지와 세부 최적화에 집중" |
| 60~79 | 양호 | 파랑 / "약점 지표 개선으로 Top Performer 진입 가능" |
| 40~59 | 보통 | 주황 / "개선이 필요한 영역이 있습니다" |
| 0~39 | 미흡 | 빨강 / "긴급 개선이 필요합니다" |

## 2. 강점/약점 분석

### 방법

**Top Performer (클러스터 0) 대비 비율**을 계산하여 상위 3개는 강점, 하위 3개는 약점으로 분류합니다.

### 비교 지표 (높을수록 좋은 6개)

| 지표 | Top Performer 기준값 |
|------|---------------------|
| 총 주문수 | 98.5건 |
| 총 매출 | R$12,161 |
| 평균 단가 | R$136 |
| 상품 다양성 | 36.3종 |
| 평균 리뷰 | 4.06점 |
| 고유 고객수 | 87.4명 |

### 계산 절차

1. 각 지표에 대해 `비율 = 셀러 값 / Top Performer 값` 계산
2. 비율이 높은 순으로 정렬
3. **상위 3개** → 강점
4. **하위 3개** → 약점

### 추가 보정: 낮을수록 좋은 지표

아래 3개 지표는 역수 비율(`Top값 / 셀러값`)로 비교합니다:

| 지표 | Top Performer 기준값 | 강점 조건 | 약점 조건 |
|------|---------------------|-----------|-----------|
| 평균 배송일 | 12.1일 | Top보다 20% 이상 짧음 | Top보다 2배 이상 김 |
| 배송 지연율 | 6% | Top보다 20% 이상 낮음 | Top보다 2배 이상 높음 |
| 저평가 비율 | 10% | Top보다 20% 이상 낮음 | Top보다 2배 이상 높음 |

## 3. 컨설팅 조언 생성 (13개 규칙)

룰 엔진이 셀러의 메트릭을 13개 규칙에 순서대로 대입하여, 조건에 해당하는 조언만 생성합니다. 생성된 조언은 **우선순위(critical → high → medium → low)** 순으로 정렬되어 표시됩니다.

### 규칙 요약표

| # | 규칙명 | 발동 조건 | 우선순위 | 핵심 근거 |
|---|--------|-----------|----------|-----------|
| 1 | 리뷰 긴급 | 평균 리뷰 < 3.5 | critical | 3.5 미만 셀러 매출 급락 (R$1,800 vs R$4,200) |
| 2 | 배송 지연 | 지연율 > 10% | critical(>20%) / high | Top Performer 지연율 6% 기준 |
| 3 | 상품 다양성 | 상품수 < 10종 | high | Top 평균 36.3종, 다양성↔매출 강한 양의 상관 |
| 4 | 사진 부족 | 평균 사진 < 2장 | high | 4장 이상 시 매출 +27.3% |
| 5 | 지역 편중 | SP 고객 비율 > 50% | medium | SP 수급비율 35 (공급 과잉) |
| 6 | 리뷰 스위트스팟 | 3.5 ≤ 리뷰 < 4.0 | high | 0.5점 개선 시 매출 +81% (R$4,200→R$7,603) |
| 7 | 카테고리 확장 | 미진출 고기회 카테고리 존재 | medium | 기회지수 상위 카테고리 비교 |
| 8 | 가격 최적화 | 평균 단가 < R$30 또는 > R$100 | low | 볼륨존(R$30~100)이 전체 거래의 45% |
| 9 | 배송일 경고 | 평균 배송일 > 20일 | high | 21일 초과 시 재구매율 1.2% (7일 이내의 1/4) |
| 10 | 저평가 원인 진단 | 리뷰 < 3.5 AND 저평가비율 > 20% | critical | 배송 vs 품질 자동 진단 |
| 11 | 취소율 관리 | 취소율 > 2% | medium(~5%) / high(>5%) | 플랫폼 노출 순위 하락 |
| 12 | 재구매 확보 | 고객 10명+ AND 재구매율 < 3% | medium | 신규 획득 비용 = 유지 비용의 5~7배 |
| 13 | 리뷰 키워드 | 특정 이슈가 리뷰의 20%+ | high | 텍스트 분석 기반 구체적 불만 분류 |

### 규칙 상세

#### 규칙 1: 리뷰 긴급 (< 3.5점)
- **조건**: `avg_review > 0` AND `avg_review < 3.5`
- **근거**: 리뷰 구간별 평균 매출 데이터
  - 1.0~2.0점: R$250 / 2.0~3.0점: R$520 / 3.0~3.5점: R$1,800
  - 3.5~4.0점: R$4,200 / **4.0~4.5점: R$7,603 (최고)**

#### 규칙 2: 배송 지연율 (> 10%)
- **조건**: `late_delivery_pct > 0.10`
- **우선순위 분기**: 20% 초과 → critical, 이하 → high
- **타겟**: 6% 이하 (Top Performer 수준)

#### 규칙 3: 상품 다양성 (< 10종)
- **조건**: `product_variety < 10`
- **비교 기준**: Top Performer 평균 36.3종

#### 규칙 4: 사진 부족 (평균 < 2장)
- **조건**: `avg_photos < 2`
- **근거**: 사진 효과 분석 데이터
  - 1장(기준) → 2장(+12%) → 3장(+20%) → 4장+(+27.3%)

#### 규칙 5: 지역 편중 (SP > 50%)
- **조건**: 고객 중 상파울루(SP) 비율 > 50%
- **추천 지역**: 수급비율 상위 3개 주(PE 178.8, BA 172.5, PA 210.0 등)

#### 규칙 6: 리뷰 스위트스팟 (3.5~4.0점)
- **조건**: `3.5 ≤ avg_review < 4.0`
- **핵심 인사이트**: 3.5~4.0 구간(R$4,200) → 4.0~4.5 구간(R$7,603)은 **+81% 매출 점프**

#### 규칙 7: 카테고리 확장 기회
- **조건**: 셀러 미진출 카테고리 중 기회지수 상위가 존재
- **기회지수 순위**: watches_gifts(7,100) > computers_accessories(5,200) > health_beauty(4,800)

#### 규칙 8: 가격 최적화
- **조건**: 평균 단가가 볼륨존(R$30~100) 밖
- **분기**: < R$30 → "가격대 상향 검토" / > R$100 → "중저가 라인 추가"

#### 규칙 9: 배송일 경고 (> 20일)
- **조건**: `avg_delivery_days > 20`
- **근거**: 배송일별 재구매율
  - 7일 이내: 4.5% / 7~14일: 3.5% / 14~21일: 2.5% / **21일 초과: 1.2%**

#### 규칙 10: 저평가 원인 진단
- **조건**: `avg_review < 3.5` AND `low_review_pct > 20%`
- **자동 진단 로직**:
  - `지연율 > 15%` OR `배송일 > 18일` → **배송 문제**로 진단
  - 그 외 → **상품 품질/기대치 불일치**로 진단

#### 규칙 11: 취소율 관리
- **조건**: `cancel_rate > 2%`
- **취소 상태**: `canceled`, `unavailable` 주문을 취소로 간주

#### 규칙 12: 재구매 고객 확보
- **조건**: `unique_customers ≥ 10` AND `repeat_customer_rate < 3%`
- **재구매 판정**: 동일 고객이 2건 이상 주문

#### 규칙 13: 리뷰 키워드 인사이트
- **조건**: 텍스트 리뷰 분석 결과, 특정 이슈가 전체의 20% 이상
- **이슈별 맞춤 액션**: 배송 지연 / 상품 품질 / 포장 문제 / 기대 불일치 각각 다른 조언

## 4. 성장 로드맵

셀러의 현재 메트릭과 목표치를 비교하여 **3단계 성장 로드맵**을 자동 생성합니다.

### 단계별 목표치

| 지표 | 단기 (1~3개월) | 중기 (3~6개월) | 장기 (6~12개월) |
|------|---------------|---------------|----------------|
| 라벨 | 기반 다지기 | 성장 가속 | Top Performer 진입 |
| 총 주문수 | 15건 | 40건 | 98건 |
| 평균 리뷰 | 3.8점 | 4.0점 | 4.1점 |
| 배송 지연율 | 8% | 5% | 4% |
| 상품 다양성 | 10종 | 20종 | 36종 |
| 평균 사진 | 3장 | - | - |
| 매출 | - | R$5,000 | R$12,000 |

### 생성 로직

- 각 단계의 목표치와 셀러 현재값을 비교
- **현재값이 목표 미달인 항목만** 로드맵에 포함
- 이미 달성한 목표는 "현재 성과 유지 및 안정화"로 표시
- 장기 목표는 모든 셀러에게 동일하게 Top Performer 수준 제시

## 5. 퍼센타일 랭킹

### 방법

전체 2,971명 셀러 대비 **상위 X%** 를 계산합니다.

### 높을수록 좋은 지표 (7개)

```
상위 퍼센타일 = (전체 셀러 중 이 셀러 이상인 비율) × 100
```

대상 지표: `total_orders`, `total_revenue`, `avg_price`, `product_variety`, `avg_review`, `unique_customers`, `items_per_order`

### 낮을수록 좋은 지표 (3개)

```
상위 퍼센타일 = (전체 셀러 중 이 셀러 이하인 비율) × 100
```

대상 지표: `low_review_pct`, `avg_delivery_days`, `late_delivery_pct`

### 해석 예시

- 매출 퍼센타일 5.2% → "전체 셀러 중 상위 5.2%"
- 배송일 퍼센타일 80.1% → "전체 셀러 중 배송이 빠른 상위 80.1%"

## 6. 맞춤 추천 진출 지역

### 개요

셀러가 현재 판매하는 카테고리를 기반으로, 셀러의 소재지를 제외한 **성장 가능성이 높은 지역 Top 5**를 추천합니다.

### 기회 점수 계산

```
기회 점수 = (수급비율 정규화 × 0.4) + (시장매출 정규화 × 0.3) + (셀러당주문 정규화 × 0.3)
```

| 요소 | 가중치 | 계산 방법 |
|------|--------|-----------|
| 수급비율 | 40% | 해당 지역의 고객수 / 셀러수 (높을수록 공급 부족 = 기회) |
| 시장매출 | 30% | 셀러의 카테고리가 해당 지역에서 발생시킨 총 매출 |
| 셀러당주문 | 30% | 해당 지역의 카테고리 총 주문수 / 경쟁 셀러 수 |

### 정규화

모든 요소는 **Min-Max 정규화 (0~100)**로 스케일링합니다:

```
정규화 값 = (값 - 최솟값) / (최댓값 - 최솟값) × 100
```

### 필터링

- 셀러의 현재 소재 주(State)는 **제외**
- 셀러가 실제 판매 중인 카테고리 데이터만 사용
- 상위 5개 지역을 기회 점수 내림차순으로 반환

### 추천 이유 자동 생성

각 추천 지역에 대해 아래 조건 해당 시 이유를 생성합니다:

## 8. 카테고리 기회 분석

### 미진출 카테고리 기회

셀러가 아직 판매하지 않는 카테고리 중 기회가 높은 것을 추천합니다.

#### 기회 점수 계산
```
기회 점수 = 셀러당 주문수 × 평균 가격 / 100
```

- `셀러당 주문수` = 해당 카테고리 전체 주문 / 셀러 수
- `평균 가격` = 해당 카테고리 평균 판매가

### 카테고리 기회지수 (벤치마크)

기회지수 = 평균 매출 × 수요 기반으로 사전 계산된 상수:

## 9. 가격 포지셔닝 & 시뮬레이션

### 가격대 분류

| 구간 | 범위 | 전체 거래 비중 |
|------|------|---------------|
| 저가 | R$0~30 | 25% |
| **볼륨존** | **R$30~100** | **45%** |
| 중가 | R$100~300 | 22% |
| 프리미엄 | R$300+ | 8% |

### 가격 포지셔닝

카테고리별 가격 통계(평균, 중앙값, P25, P75)를 계산하고 셀러의 평균 단가가 어디에 위치하는지 표시합니다.

### 매출 시뮬레이션

특정 카테고리+지역에서 가격대별 예상 월 매출을 추정합니다:

```
데이터 기간(월) = (마지막 주문일 - 첫 주문일) / 30
해당 지역 월 주문수 = 지역 내 카테고리 주문수 / 데이터 기간
가격대별 주문 비율 = 해당 가격대 주문수 / 전체 주문수
예상 월 주문 = 월 주문수 × 가격대별 주문 비율
예상 월 매출 = 예상 월 주문 × 해당 가격대 평균 가격
```

## 10. 크로스셀 카테고리 추천

### 방법

1. 셀러와 **같은 카테고리를 판매하는 다른 셀러들**을 찾음
2. 그 셀러들이 **추가로 판매하는 다른 카테고리**를 집계
3. **채택률(adoption_rate)** 계산:
   ```
   채택률 = 해당 카테고리를 판매하는 유사 셀러 수 / 전체 유사 셀러 수
   ```
4. 채택률 5% 이상인 카테고리만 필터링
5. 채택률 높은 순으로 상위 10개 추천

### 해석

- 채택률 30% → "유사 셀러의 30%가 이 카테고리도 판매 중"
- 채택률이 높을수록 시너지 가능성이 큰 카테고리

## 11. 리뷰 키워드 분류

### 개요

포르투갈어 리뷰 텍스트를 **키워드 매칭**으로 4개 이슈 카테고리로 자동 분류합니다.

### 이슈 카테고리별 키워드

| 카테고리 | 주요 키워드 (포르투갈어) | 의미 |
|----------|------------------------|------|
| **배송 지연** | entrega, atraso, demora, prazo, correios, extraviado | 배송 관련 불만 |
| **상품 품질** | qualidade, defeito, quebrado, ruim, não funciona | 제품 결함/불량 |
| **포장 문제** | embalagem, amassado, rasgado, mal embalado | 포장 파손 |
| **기대 불일치** | foto, descrição, tamanho, cor, diferente | 상품 설명과 실물 차이 |

### 긍정 키워드

`excelente`, `ótimo`, `perfeito`, `recomendo`, `rápido`, `antes do prazo` 등

### 분류 프로세스

1. 리뷰 텍스트를 소문자로 변환
2. 각 카테고리의 키워드 목록과 매칭 (부분 문자열 포함)
3. **하나의 리뷰가 복수 카테고리에 해당 가능**
4. 1~2점 리뷰는 별도로 `negative_issues`로 분류

### 컨설팅 규칙 연동 (규칙 13)

- 가장 빈번한 이슈(primary_issue)가 전체 텍스트 리뷰의 **20% 이상**일 때 조언 생성
- 이슈별 맞춤 개선 액션 제공

## 부록: 주요 메트릭 계산 방법

### KPI 메트릭 (preprocessor.py)

| 메트릭 | 계산 방법 |
|--------|-----------|
| 총 매출 | `seller_data["price"].sum()` |
| 총 주문수 | `seller_data["order_id"].nunique()` |
| 고유 고객수 | `seller_data["customer_unique_id"].nunique()` |
| 평균 주문가치(AOV) | 총 매출 / 총 주문수 |
| 상품 다양성 | `seller_data["product_id"].nunique()` |
| 평균 단가 | `seller_data["price"].mean()` |
| 평균 리뷰 | `review_score.mean()` |
| 저평가 비율 | `(review_score <= 2).mean()` |
| 평균 배송일 | 배송완료 주문의 `delivery_days.mean()` |
| 배송 지연율 | 배송완료 주문의 `is_late.mean()` |
| 취소율 | `canceled + unavailable 주문수 / 전체 주문수` |
| 재구매율 | `2건 이상 주문 고객수 / 전체 고유 고객수` |
| 평균 사진수 | `product_photos_qty.mean()` |

### 셀러-고객 거리 (Haversine)

```
거리(km) = 6371 × 2 × arcsin(√(sin²(Δlat/2) + cos(lat1)·cos(lat2)·sin²(Δlng/2)))
```

- 셀러 우편번호 → geolocation 위경도
- 고객 우편번호 → geolocation 위경도
- 거리 구간: 0~200km / 200~500km / 500~1000km / 1000~2000km / 2000km+

### 카테고리 내 순위

셀러의 상위 3개 카테고리에 대해, 같은 카테고리를 판매하는 전체 셀러 중 매출 순위와 리뷰 순위를 계산합니다.

```
매출 순위 = 해당 카테고리 셀러 중 이 셀러 이상 매출인 셀러 수
리뷰 순위 = 해당 카테고리 셀러 중 이 셀러 이상 리뷰인 셀러 수
```
